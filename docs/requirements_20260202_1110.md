修正仕様書
個別画面 → メイン画面 D&D 時の選択状態制御
＋ 個別画面ページの拡大表示仕様
1. 背景・問題点

現在の実装では、**個別画面（ページビュー）からメイン画面（ファイルグリッド）へドラッグ＆ドロップ（D&D）**した際に、

ドロップ先の PDF ファイルが選択状態になる

しかし、ドロップ前にメイン画面で選択されていた他の PDF ファイルの選択状態が解除されない

という挙動になっている。

一方で、

メイン画面 → 個別画面 の D&D では

移動先のみが選択状態となり

それ以外の選択は解除される

という、期待される挙動がすでに実装されている。

操作の一貫性・UX の観点から、両者の挙動を統一する必要がある。

また、個別画面では
ページ内容を詳細に確認するための拡大表示手段が不足している。

2. 修正方針（結論）

個別画面 → メイン画面 D&D 時も、メイン → 個別と同じ選択状態制御ルールを適用する

本操作に関連する split 機能は廃止（削除）する

個別画面でページをダブルクリックした場合は、拡大表示用の表示モードを提供する

3. 修正後の仕様（D&D）
3.1 対象操作

個別画面（ページ単位表示）から

メイン画面（ファイルグリッド）への

ドラッグ＆ドロップ操作

3.2 選択状態の制御ルール（修正後）

D&D が成立した時点で、以下を必ず満たすこと。

ドロップ先の PDF ファイル

選択状態（selected = true）にする

ドロップ前にメイン画面で選択されていた PDF ファイル

すべて選択解除（selected = false）する

ただし、ドロップ先と同一のファイルは除外する

結果として、

メイン画面では
「ドロップ先の PDF ファイルのみが選択状態」
となること

4. Split 機能の扱い（本修正での変更点）

個別画面 → メイン画面 D&D に関連する split 機能は完全に削除する

D&D 操作によって、

ページが分割されて新規 PDF が生成される

split 専用の分岐処理が実行される
といった挙動は 一切行わない

本操作は 常に統合（移動／マージ）系の D&D として扱う

5. 個別画面：ページ拡大表示仕様（追加）
5.1 対象操作

個別画面（ページ一覧表示）において

特定のページをダブルクリックした場合

5.2 表示挙動

ダブルクリックされたページを
拡大表示用の表示状態で表示する

拡大表示時は、

サムネイルサイズではなく

**ページ本来の通常表示サイズ（可読サイズ）**で表示する

表示対象は ダブルクリックされた 1 ページのみとする

5.3 拡大縮小（ズーム）仕様

拡大表示状態では、以下を可能とする。

ページの 拡大・縮小操作

マウスホイール

スライダー

ボタン
など、実装しやすい手段で可

ズーム倍率は連続的に変更できること

最低倍率・最大倍率は UI 的に破綻しない範囲で定義する

5.4 表示モードの位置づけ

この拡大表示は
「編集モード」や「分割モード」ではない

ページ内容を確認するための
閲覧専用の表示モードとして扱う

拡大表示中であっても、

ページ構造

PDF の統合状態
などは変更しない

6. 明示的に行わないこと（非仕様）

個別画面側のページ選択状態を変更する必要はない

拡大表示中にページ編集・分割・削除は行わない

複数ページを同時に拡大表示する仕様は採用しない

Split 機能の代替仕様や再設計は本修正では行わない

Undo / Redo の粒度や意味は本修正では変更しない
（D&D 全体で 1 ステップである点は従来通り）

7. 既存仕様との対応関係
7.1 選択状態
操作方向	選択状態の扱い
メイン → 個別	移動先のみ選択、他は解除（既存）
個別 → メイン	移動先のみ選択、他は解除（本修正）
7.2 表示モード
操作	表示
個別画面 通常表示	ページサムネイル一覧
ページ ダブルクリック	単ページ拡大表示（ズーム可）
8. 実装上の注意点（参考）

D&D 完了ハンドラ内で

clear_selection(except=drop_target)
のような明示的な選択解除処理を行うこと

UI 状態（選択・ズーム倍率）と内部状態の同期を必ず取ること

split 関連の分岐・フラグ・UI 表示が残っていないかを確認すること

拡大表示は 既存ページビューを破壊せず、状態遷移として実装すること

以上。
