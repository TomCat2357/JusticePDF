## 問題1: Redo連続実行時に Qt Widget が既に削除されている例外が出る

### 再現条件
- メイン画面で並び替えをUndo/Redo
- Redoを連続実行すると以下が発生
  RuntimeError: wrapped C/C++ object of type PDFCard has been deleted

### 原因の前提
- PDFCard(QWdiget) をUndo/Redoの状態として直接保持・再利用している
- _refresh_grid() 内で deleteLater() または親Widget変更が起きている
- Redo時に「既にC++側で破棄されたWidget」を addWidget しようとしている

### 修正方針（重要）
- Undo/Redoの履歴に **Qt Widget (PDFCard) を一切保持しない**
- Undo/Redoで保持するのは以下のみとする：
  - PDFファイルID / UUID
  - 並び順（list[str] などの純データ）
- Redo/Undo 実行時は
  1. 既存のGrid上のWidgetをすべて安全にクリア
  2. モデル（ファイル順序）を復元
  3. **必ずPDFCardを新規生成してGridに追加**

### 実装ルール
- UndoAction に QWidget や PDFCard を入れてはいけない
- Grid再描画は「毎回再生成」が原則
- `sip.isdeleted(obj)` や `obj.parent() is None` に依存する設計は禁止
- Widget寿命は View が完全に管理し、UndoManager は Model のみを扱う

### 期待する成果
- Redoを100回連続実行しても例外が出ない
- PDFCardのライフサイクルが View のみで完結する


## 問題2: 個別画面からページをメイン画面へ移動後、個別画面の表示が更新されない

### 再現条件
- 個別画面（ページ一覧）からページをメイン画面へD&D
- データ上はページが移動している
- しかし個別画面にはそのページが残ったまま表示される

### 前提
- Model（PDFPageList 等）は正しく更新されている
- 問題は View の即時同期不足

### 修正方針
- 「ページ移動」は以下を**必ず同時に行う操作**と定義する：
  1. 元PDFの page list から remove
  2. 先PDFへ page を add
  3. **元の個別画面Viewを即時 refresh**
- Model更新だけで View が変わる前提は禁止

### 実装ルール
- ページ移動完了時に、以下のいずれかを必ず行う：
  - detail_view.refresh_pages()
  - または pageMoved シグナルを emit → 個別画面が再描画
- D&Dハンドラ内で View 更新をスキップしてはいけない
- 「画面を閉じたら直る」は不具合として扱う

### 期待する挙動
- ページ移動完了と同時に、個別画面からそのページが消える
- 再描画の遅延・一貫性ズレがない
