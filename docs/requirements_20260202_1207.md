# 追加仕様書（JusticePDF UI / D&D 挙動 追補）
Version: 1.1  
Date: 2026-02-02  
対象アプリ名: JusticePDF  

---

## 1. アプリケーション名・ウインドウタイトル

### 1.1 変更内容

- アプリケーションの正式名称を **JusticePDF** とする
- OS 上で表示されるすべてのウインドウタイトルを **JusticePDF** に統一する

### 1.2 対象ウインドウ

| ウインドウ種別 | 現状 | 変更後 |
|---|---|---|
| メインウインドウ | PDFas | JusticePDF |
| 個別画面（ページ編集画面） | PDFas | JusticePDF |
| 拡大表示ウインドウ（存在する場合） | PDFas | JusticePDF |
| タスクバー / Alt+Tab 表示 | PDFas | JusticePDF |

### 1.3 実装指針

- `QMainWindow.setWindowTitle()` 等で明示的に **JusticePDF** を設定する
- ファイル名や状態（例: `JusticePDF - sample.pdf`）を付加する場合も、先頭は必ず `JusticePDF` とする

## 2. 個別画面 → メイン画面 D&D 時の「間」ハイライト表示

### 2.1 背景

現在の実装では、

- **メイン画面 → メイン画面**
  - PDFファイル同士の「間」にドラッグした際、挿入位置が視覚的にハイライトされる
- **個別画面 → メイン画面**
  - PDF上に重ねた場合は対象PDFがハイライトされる
  - **PDFファイル同士の「間」にドラッグした場合、視覚的な変化が起きない**

という不整合が存在する。

---

## 2.2 追加仕様（必須）

### 2.2.1 対象操作

- **個別画面（ページビュー）から**
- **メイン画面（ファイルグリッド）へ**
- ページ（1ページまたは複数ページ）をドラッグ＆ドロップする操作

---

### 2.2.2 挿入位置の視覚フィードバック

#### A. ファイル間にドロップ可能な場合

- ドラッグ中カーソルが
  - メイン画面の PDF ファイル **とファイルの間**
  - 挿入判定エリア（Insert 判定）に入った場合

👉 **その「間」の背景色、または挿入ラインの色を変更する**

- 表示例（いずれか、または併用可）
  - 縦ライン / 横ラインの強調表示
  - 間の背景色ハイライト
  - 挿入位置を示すインジケータ表示

#### B. 表示ルール

| 状態 | 表示 |
|---|---|
| ドロップ不可 | 表示なし |
| ファイル上（Overlay 判定） | 対象PDF全体をハイライト |
| ファイル間（Insert 判定） | **間の色を変更して明示** |

---

### 2.2.3 メイン → メイン と同等の UX

- **メイン画面 → メイン画面** の D&D で実装されている
  - ファイル間ハイライト
  - 挿入位置可視化
- 上記と **同一ロジック・同一見た目** を原則とする

※ D&D の「データ種別」が  
`PDFCARD_MIME_TYPE`（ファイル）か  
`PAGETHUMBNAIL_MIME_TYPE`（ページ）か  
の違いによって、視覚フィードバックに差が出てはいけない。

---

## 2.3 判定ロジック（整理）

### 2.3.1 判定優先順位

1. **カーソル位置が PDF 上**
   - Overlay 判定
   - 対象 PDF をハイライト
2. **カーソル位置が PDF と PDF の間**
   - Insert 判定
   - **間をハイライト**
3. それ以外
   - ハイライトなし

---

### 2.3.2 操作結果（参考）

| 操作 | 結果 |
|---|---|
| 個別 → メイン（間にドロップ） | 指定位置にページを挿入 |
| 個別 → メイン（PDF上にドロップ） | ページを既存PDFに統合 |

※ 処理仕様自体は既存仕様に準拠し、**本追加仕様は視覚フィードバックの統一を目的とする**

---

## 3. 非対象事項（本仕様では扱わない）

- Undo / Redo の内部実装変更
- ページ統合順序の変更
- D&D データ構造（Payload）の変更
- 判定閾値（中央◯% / 端◯%）の数値変更

---

## 4. 目的まとめ

本追加仕様の目的は以下に集約される。

- アプリ名を **JusticePDF** に正式統一すること
- D&D 操作における
  - **メイン→メイン**
  - **個別→メイン**
  の UX を完全に揃え、
- 「落とせる場所が直感的に分かる」操作感を保証すること
