# PDFスタック整理アプリ 追加仕様書
Version: 1.1  
Date: 2026-02-01  
対象: メイン画面 / 個別画面 / 操作履歴 / UI表現

本仕様書は、既存の「PDFスタック整理アプリ 仕様（Version 1.0）」に対する**追加仕様**である。

---

## 1. ドラッグ＆ドロップ時の視覚フィードバック

### 1.1 メイン画面（File Grid）での挙動

#### 1.1.1 PDF間へのドラッグ
- メイン画面で PDF（ファイル）またはページをドラッグ中、
- **PDFとPDFの間**にドラッグカーソルが入った場合：
  - 挿入位置を示すため、**その「間」が強調表示**される。
  - 表示方法の例：
    - 縦ラインのハイライト
    - 間の背景色変更
- この「間のハイライト」は **メイン画面限定**の挙動とする。

#### 1.1.2 他のPDF上へのドラッグ
- メイン画面で、ドラッグ中のオブジェクトが **既存のPDFファイル上に重なった場合**：
  - 対象となる PDF 全体の背景色または枠線色を変更する。
  - これは「重ね合わせ（マージ）」可能であることを示す。

#### 1.1.3 視覚状態の排他ルール
- 「間への挿入ハイライト」と「PDF上のハイライト」は同時に表示しない。
- カーソル位置に応じて、いずれか一方のみを表示する。

---

### 1.2 個別画面（Detail / Page View）での挙動

- 個別画面でページをドラッグした場合：
  - メイン画面上では **1.1 のルールをそのまま適用**する。
- 個別画面内では：
  - ページ間にドラッグした場合、ページ間の挿入位置をハイライトする。
  - ページ上にドラッグした場合、そのページが選択対象であることを示す。

## 2. ダブルクリック操作

### 2.1 メイン画面でのダブルクリック

- メイン画面で PDF ファイルを **ダブルクリック**した場合：
  - 対応する個別画面（ページビュー）を **別ウインドウ（別画面）**として開く。
- 挙動の要件：
  - メイン画面は閉じない。
  - 複数の個別画面を同時に開くことが可能。

---

## 3. 操作履歴（Undo / Redo）

### 3.1 基本仕様

- アプリ起動から終了までの間、**最大 100 ステップ分の操作履歴**を保持する。
- 対応操作：
  - Undo：`Ctrl + Z`
  - Redo：`Ctrl + Y`

### 3.2 ステップ定義

- **1ユーザー操作 = 1ステップ**とする。
- 以下はすべて **同一ルール**で履歴管理する：
  - メイン画面での操作
  - 個別画面での操作

#### ステップに含まれる操作例
- PDFファイルの並び替え
- PDFの重ね合わせ（マージ）
- ページの移動・挿入・統合
- 個別画面でのページ順変更
- メイン ⇔ 個別画面間の D&D 操作

### 3.3 Undo / Redo の動作ルール

- Undo 実行時：
  - 直前の 1 ステップ分の操作を完全に巻き戻す。
- Redo 実行時：
  - Undo で戻した操作を再適用する。
- 新しい操作を行った場合：
  - Redo 用の履歴は破棄される（一般的なエディタ挙動）。

## 4. UI / デザイン改善方針

### 4.1 課題認識
- 現状の画面は「もっさりしている」ため、操作感・視認性の改善を行う。

### 4.2 デザイン方針

- 全体方針：
  - フラットで軽量な UI
  - 操作に対する即時レスポンスを重視
- 具体方針：
  - 余白を適切に取り、情報密度を下げすぎない
  - 不要な枠線・装飾を削減
  - ホバー・ドラッグ時のアニメーションは **短く・軽く**
  - 色数は抑え、状態変化（選択・挿入・重ね合わせ）が直感的に分かる配色とする

### 4.3 パフォーマンス配慮
- ドラッグ中の再描画やハイライト処理は最小限に抑える。
- ページ数・PDF数が多い場合でも、操作遅延を体感させないことを重視する。
